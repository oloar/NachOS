26 // The following class defines the Nachos "file header" (in UNIX terms,
27 // the "i-node"), describing where on disk to find all of the data in the file.
28 // The file header is organized as a simple table of pointers to
29 // data blocks.
30 //
31 // The file header data structure can be stored in memory or on disk.
32 // When it is on disk, it is stored in a single sector -- this means
33 // that we assume the size of this data structure to be the same
34 // as one disk sector. Without indirect addressing, this
35 // limits the maximum file length to just under 4K bytes.
36 //
37 // There is no constructor; rather the file header can be initialized
38 // by allocating blocks for the file (if it is a new file), or by
39 // reading it from disk.
